# 1. 文書概要

本書は、
低コストで小規模に開始できるクラウドサービス活用型の
学習・検証プラットフォームについて、
概念設計を整理することを目的とする。

本書では、

* どのような考え方でシステムを構成するか
* どのような用途に利用するか
* 将来的にどの方向へ発展させるか

を整理する。

# 2. システムコンセプト

本システムは、
クラウドサービスを利用して、
低コストで小さく始められる技術検証用プラットフォームを構築することを目的とする。

社内向けツールや小規模な Web サービスでは、
大規模なインフラ構成を必要としない一方で、

* Web画面を提供したい
* APIで処理を受け付けたい
* メッセージングを利用したい
* データを保存したい
* 将来的に機能拡張できる余地を残したい

という要求が発生する。

本システムでは、
このような小規模システムを、
できるだけ低コストかつ低運用負荷で構築・検証できる
技術検証用箱庭（Sandbox）として整理する。

本書の主眼は、
特定サービスの使い方ではなく、
クラウドサービスを活用して小規模なイベント駆動型システムをどのように構成するか、
その考え方を整理することにある。

# 3. 背景

近年は、

* Web API
* SaaS
* IoT
* Webhook
* イベント通知

など、
イベント駆動型のシステム構成が増えている。

しかし、
学習用途や小規模開発向けに、

* API
* メッセージング
* データ保存

を含む構成を、
低コストで試せる環境は限られている。

本システムでは、
そのようなイベント駆動型システムを
小規模構成で再現する。

# 4. 技術検証用箱庭

本システムは、
単なるサンプルプログラムではなく、
技術を試し、
設計を整理し、
将来の開発へ流用するための
技術検証用箱庭として位置付ける。

また、
作成した設計資料についても、

* 技術検証
* 個人開発
* 小規模業務システム

に流用可能なテンプレートとして整備する。

# 5. 想定利用用途

本システムは以下用途を想定する。

* クラウドサービスを利用した Web サービス構成の学習
* SaaS / PaaS を利用した低コスト構成の検証
* 小規模業務アプリケーションの試作
* イベント駆動型システムの学習
* メッセージング処理の学習
* 個人開発・技術検証
* 将来案件向けの設計テンプレート整備

# 6. 想定しない内容

本システムでは、
以下内容は対象外とする。

* 本番運用
* 高可用性構成
* 大規模システム
* 高度認証
* 大量アクセス対応

# 7. 将来構想

本システムをベースとして、
以下構成への発展を想定する。

* IoT連携
* PLC連携
* Webhook連携
* リアルタイム監視
* AI連携
* 通知機能

# 8. まとめ

本システムは、
低コストで小規模に開始できるイベント駆動型システムの
技術検証用箱庭として構築する。

また、
作成した設計資料は、
将来の技術検証や小規模システム開発時に再利用できる
テンプレートとして整備する。

本書をベースとして、
後続の基本仕様書および内部設計書を作成する。
