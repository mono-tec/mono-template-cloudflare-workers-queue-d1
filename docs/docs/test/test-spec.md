# 1. 文書概要

本書は、
Cloudflare Workers Queue + D1 サンプルシステムの
テスト仕様を定義することを目的とする。

本書では、
テスト計画書で定義したテスト方針に基づき、
具体的なテスト項目、手順、期待結果を整理する。

# 2. テスト対象

本テストでは、
以下機能を対象とする。

| No | 対象          | 内容                |
| -- | ----------- | ----------------- |
| 1  | Web UI      | イベント送信、件数表示、一覧表示  |
| 2  | Workers API | イベント受付、件数取得、一覧取得  |
| 3  | Queue       | イベント登録、Consumer連携 |
| 4  | D1 Database | イベント保存、件数取得、一覧取得  |

# 3. 前提条件

テスト実施前に、
以下状態であることを確認する。

| No | 前提条件                                   |
| -- | -------------------------------------- |
| 1  | Cloudflare Workers が Deploy 済みであること    |
| 2  | Cloudflare Queue が作成済みであること            |
| 3  | Cloudflare D1 Database が作成済みであること      |
| 4  | wrangler.jsonc に必要な Binding が設定済みであること |
| 5  | Web画面へアクセス可能であること                      |

## 5.3 API-001 イベント送信API正常

| 項目    | 内容                              |
| ----- | ------------------------------- |
| テストID | API-001                         |
| テスト対象 | Workers API                     |
| テスト内容 | POST /api/events が正常応答することを確認する |

### Request

```json
{
  "eventType": "button_click",
  "message": "test event",
  "payload": {
    "source": "test"
  }
}
```

```json
{
  "success": true,
  "message": "Event queued."
}
```

### 期待結果

* HTTP 200 が返却される
* Queueへイベント登録される

## 5.9 DB-001 イベント登録

| 項目    | 内容                           |
| ----- | ---------------------------- |
| テストID | DB-001                       |
| テスト対象 | D1 Database                  |
| テスト内容 | event_log にイベントが登録されることを確認する |

### 確認SQL

```sql
SELECT
    id,
    event_id,
    event_type,
    message,
    created_at
FROM
    event_log
ORDER BY
    id DESC
LIMIT 1;
```

### 期待結果

* event_log に1件登録されている
* event_type が `button_click` である
* message が送信値と一致する

## 5.10 結合テスト IT-001 イベント送信から一覧表示まで

| 項目    | 内容                        |
| ----- | ------------------------- |
| テストID | IT-001                    |
| テスト対象 | UI / API / Queue / D1     |
| テスト内容 | イベント送信から一覧表示までの一連の流れを確認する |

### 手順

1. イベント監視画面を表示する
2. 現在のイベント件数を確認する
3. イベント送信ボタンを押下する
4. 数秒待機する
5. イベント件数を確認する
6. 最新イベント一覧を確認する

### 期待結果

* イベント件数が増加する
* 最新イベント一覧に送信イベントが表示される
* API / Queue / D1 が連携している

# 6. 合格基準

以下をすべて満たした場合、
テスト合格とする。

* UIテストがすべて成功する
* APIテストがすべて成功する
* Queueテストがすべて成功する
* Databaseテストがすべて成功する
* 結合テストが成功する
